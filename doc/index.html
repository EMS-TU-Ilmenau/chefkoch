Architecture¶

The architecture is described from two points of view. The first explanation is given with regards to the functionality of the system. The second explanation describes the structure of the software architecture divided into frontend, backend and database.

Chefkoch is a tool to manage the computing process of simulations that are often repeated with only small changes in input or parameter data. Starting the simulations and collecting the results manually is a big threat to consistence as it is easy to falsely use wrong parameters or old intermediate results. Also, it is hard to decide which parts of the simulation need to be repeated and which parts are not affected by a change to only a few parameters. There is the potencial to save computing time.

The project name Chefkoch which means chef cook evolved from the following conceptual idea. Chefkoch is the coordinator of the simulation computing process and works with different parallel threads as a chef works with kitchen assistants.

The picture shows the components of the computing process. The fridge is a symbol for the database as it stores ingredients and prepared dishes. The kitchen is the processing unit, e.g. a processor, a core or a computer. This processing unit is run by a kitchen assistant, refering to a process running on an machine or an individual thread. The assistants get their orders from the chef cook as the threads are started by Chefkoch. These orders depend on the meal. The chef knows which ingredients are needed for a meal and which steps are required to cook it. The chef also knows the flavour preferences of the guest. Not every ingredient is needed for every step in cooking, so the assistant only gets a list of relevant ingredients and a note concerning the flavour. After an assistant is done, the prepared food goes into the fridge for another assistant to use. There is more than one meal ordered. So the chef works parallel on all of them.
